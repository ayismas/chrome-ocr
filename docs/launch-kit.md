# Launch Kit

Use this file as the maintainer-facing checklist for turning the repository into a post people will actually share.

## One-line positioning

- Turn local PDFs and images into Markdown with Chrome's hidden OCR engine.
- Offline OCR with no API key, no cloud bill, and no model download.
- A Windows OCR tool that reuses Chrome's own production Screen AI component.

## What to demo first

- A scanned PDF turning into readable Markdown in one command.
- A mixed-layout page where headings and tables survive extraction.
- The `chrome-ocr doctor` command confirming local setup.
- A benchmark table generated by `benchmarks/run_benchmark.py`.

## Assets to collect before posting

- One terminal recording of `chrome-ocr pdf report.pdf -o report.md`
- One before/after screenshot for a scanned page
- One benchmark table from `benchmarks/README.md`
- One short explanation of why using Chrome's bundled OCR is unusual

## Suggested post titles

- I turned Chrome's hidden OCR engine into a local Python tool
- Local PDF and image OCR with Chrome's Screen AI, no API key required
- chrome-ocr: offline Markdown extraction powered by Chrome's bundled model

## Release post

```text
chrome-ocr is an open-source Python tool that turns local PDFs and images into Markdown by calling Chrome's built-in Screen AI engine.

- no API key
- no cloud round-trip
- no separate model download
- layout-aware output for headings, paragraphs, tables, and formulas

Repo: https://github.com/ayismas/chrome-ocr
```

## Posting order

1. Push a release-quality README and benchmark table.
2. Publish a GitHub release with one screenshot and one benchmark.
3. Post to GitHub first, then Hacker News / Reddit / forums, then short-form social.
